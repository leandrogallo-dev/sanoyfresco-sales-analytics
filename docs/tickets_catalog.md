# Data Catalog – Tickets Table

## Overview
The **tickets table** stores transactional sales data generated by the retail system.  
Each record represents a **product line within a customer order**, containing information about the customer, product, department, quantities, and total transaction value.

This dataset is used for **sales analytics, customer behavior analysis, and product performance evaluation**.

Typical use cases include:

- Revenue analysis
- Product sales performance
- Customer purchasing behavior
- Department and section sales metrics
- Order value analysis

---

## Table: **dbo.tickets**

**Purpose:**  
Stores detailed sales transactions for each product sold in the store.  
Each row represents a **product item within an order (ticket)** and includes information about the product, quantity sold, and total revenue generated.

---

### Columns

| Column Name      | Data Type     | Description |
|------------------|---------------|-------------|
| **id_pedido**    | INT NOT NULL | Unique identifier for each order or purchase transaction. Multiple rows may share the same order ID if multiple products were purchased in the same order. |
| **id_cliente**   | INT NOT NULL | Unique identifier of the customer who made the purchase. Used to analyze customer behavior and spending patterns. |
| **fecha**        | DATE | Date when the transaction was recorded. Used for time-based analysis such as daily, monthly, or yearly revenue trends. |
| **hora**         | INT | Hour of the day when the transaction occurred. Useful for analyzing peak sales hours. |
| **id_departamento** | INT NOT NULL | Identifier of the department where the product belongs (e.g., groceries, beverages, bakery). Used to analyze department-level performance. |
| **id_seccion**   | INT NOT NULL | Identifier of the section within the department where the product is categorized. Enables more granular sales analysis. |
| **id_producto**  | INT NOT NULL | Unique identifier of the product sold. |
| **nombre_producto** | NVARCHAR(255) | Name or description of the product sold in the transaction. |
| **precio_unitario** | DECIMAL(18,2) | Price of a single unit of the product at the time of the transaction. |
| **cantidad**     | INT | Number of units of the product purchased in the order. |
| **precio_total** | DECIMAL(18,2) | Total amount generated by the product in the transaction. Usually calculated as `precio_unitario * cantidad`. |

---

## Data Granularity
The table is stored at the **transactional product level**, meaning:

- One order (`id_pedido`) may contain multiple products.
- Each row represents **one product within an order**.

Example:

| id_pedido | id_producto | cantidad |
|-----------|-------------|----------|
| 1001      | 50          | 2        |
| 1001      | 78          | 1        |

This means order **1001** contained **two different products**.
